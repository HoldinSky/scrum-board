How it works

This web application is made for those who admire to improve their workflow. It is proved that people do their job the best when nothing disturbs them and SCRUM board will help You with that.

If You are wondering what the SCRUM is and how its contribution improves Your life, You might visit SCRUM's official page and learn more about it.

Sprint

Whole developing process of any product consists of equally divided spans of time - sprints, usually a couple of weeks, maximum - a month long, during which all attention is drawn to limited amount of tasks.

Each sprint has its own backlog that is filled with all the planned tasks. You can pick whatever task You want to then complete it. Only one task can be picked at a time, so no disturbing thoughts will overwhelm You. When the task is ready You should manually mark it as that.

Board configuration

Before the work is started the sprint must be configured. In the Configure board section the duration of each sprint and start of the first one can be set.

In addition Project's backlog can be found there. The part that will hold all the planned features for the future project. Every new well-planned task should be placed here. Afterwards before the start of new sprint tasks must be manually picked to be placed in the backlog of the sprint. Although all uncompleted tasks of previous sprint automatically transfer to the next one.

Task description

While creating task in the project's backlog only its description must be set. However, when things come to the development before any task can be picked its difficulty must be set. It is recommended that the team set it by themselves based on everyone's opinion. Difficulty then will help to define team's velocity which measures average amount of tasks completed with different difficulties.

When the task is chosen by developer, the requisites will be automatically attached to it, however it is possible to invite others to the group of responsible for the task as well as developer can join others on any task with the confirmation of those who are already busy with it. Start time of the task is set automatically as the moment of pressing the Start button on the task. The same goes with the finish of the task.

No-sprint tasks

If You do not belong to any developers team You still can use helpful part of the application in the section no-sprint tasks. It allows You to create Your own backlog of todo list and then work with it like with the team's board.
